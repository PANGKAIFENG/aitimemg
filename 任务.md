📂 全局规范进化体系：背景与目标备忘录
1. 项目背景 (Background)
在开发 时间管理 (Temporal) 项目的过程中，随着功能复杂度增加（如多泳道时间轴、复杂拖拽交互），出现了多次 “功能退化 (Regression)”。

典型案例：修复 A Bug 时引入了 B Bug；由于对 DOM 选择器唯一性考虑不足，导致在多泳道场景下拖拽计算失效；由于 TypeScript 类型导入规范不严，导致编译正常的代码在运行时白屏。
痛点分析：现有的项目规范侧重于“视觉风格”和“文档格式”，缺乏**“工程稳定性”和“防退化逻辑”**的长效沉淀。
2. 系统目标 (Objectives)
跨项目复用：建立一套独立于具体项目的 “元规范库 (Meta-standards)”，让所有项目继承相同的底层工程逻辑。
持续进化 (Logic Evolution)：建立 “问题 -> 复盘 -> 抽象 -> 规则合入” 的闭环。项目在开发中踩的坑，要立刻转化为全局规则，防止在其他项目中重复犯错。
AI 深度协同：确保所有主流 AI 编程工具（Antigravity, Cursor, Claude CLI 等）都能实时读取并强制执行这套全局规则。
