# Temporal 时间管理项目 - 数据库/后端服务选型调研

> 调研时间：2025-12-21
> 调研人：AI 助手
> 状态：初步调研，待收集更多建议

---

## 一、项目背景

### 项目简介
Temporal 是一个基于四象限法则的时间管理应用，帮助用户按「重要/紧急」维度管理任务。

### 技术栈
- 前端：React 19 + TypeScript + Vite
- UI：Ant Design 6
- 状态管理：Zustand
- 数据请求：TanStack React Query

### 当前状态
- 四象限视图已完成开发
- 数据暂存于浏览器 localStorage
- 需要选择后端服务实现数据持久化和多端同步

---

## 二、核心诉求

### 目标用户
- **主要市场：中国大陆**
- 初期用户：个人使用 + 少量朋友（约 3-5 人）
- 中期规划：面向国内个人用户推广

### 技术要求

| 要求 | 优先级 | 说明 |
|------|--------|------|
| 国内访问速度 | ⭐⭐⭐ 高 | 延迟需控制在可接受范围（<100ms 最佳） |
| 开发成本 | ⭐⭐⭐ 高 | 个人开发者，希望尽量少改代码 |
| 运维成本 | ⭐⭐⭐ 高 | 无专职运维，最好是托管服务 |
| 费用 | ⭐⭐ 中 | 初期免费或低成本，后期可接受合理付费 |
| 扩展性 | ⭐ 低 | 初期用户量小，暂不考虑高并发 |

### 功能需求

- [x] 数据库存储（任务 CRUD）
- [x] 用户认证（邮箱/手机/微信登录）
- [ ] 实时同步（可选，多端同步时需要）
- [ ] 文件存储（可选，后期可能需要）

## 三、候选方案调研

### 方案一：Supabase 海外节点（新加坡）

**简介**：Supabase 是开源的 Firebase 替代品，提供 PostgreSQL 数据库、认证、存储、实时订阅等功能。

| 维度 | 评估 |
|------|------|
| 国内延迟 | ❌ 200ms+，高峰期更高 |
| 稳定性 | ⚠️ 有被墙风险，需自定义域名 |
| 开发体验 | ✅ 优秀，文档完善 |
| 价格 | ✅ 免费版：500MB 数据库、1GB 存储、50K 月活 |
| 迁移成本 | - 基准方案 |

**结论**：国内访问体验差，不推荐作为中国市场主力方案。

**参考**：
- [Supabase 官网](https://supabase.com/)
- [知乎：符合国情的 Supabase 私有化部署](https://zhuanlan.zhihu.com/p/633443402)

---

### 方案二：MemFire Cloud（Supabase 国内版）⭐ 推荐

**简介**：MemFire Cloud 基于开源 Supabase 构建，专为国内开发者优化，API 100% 兼容 Supabase。

| 维度 | 评估 |
|------|------|
| 国内延迟 | ✅ 国内节点，延迟极低 |
| 稳定性 | ✅ 无被墙风险 |
| 开发体验 | ✅ 与 Supabase 一致 |
| 价格 | ✅ 免费版可用于个人项目 |
| 迁移成本 | ✅ 几乎为零（仅换 URL 和 Key） |

**核心功能**：
- PostgreSQL 云数据库
- RESTful API 自动生成
- 用户认证（支持微信登录）
- 对象存储
- 实时数据库订阅

**优势**：
- 提供 Vue/React/uni-app/微信小程序脚手架
- 兼容国内开发生态
- 数据存储在国内，合规

**结论**：**最推荐方案**，兼顾开发体验和国内访问速度。

**参考**：
- [MemFire Cloud 官网](https://memfiredb.com/)
- [MemFire Cloud 价格页](https://memfiredb.com/price.html)
- [CSDN：Supabase 国内版介绍](https://blog.csdn.net/whdxlxl/article/details/136469432)

---

### 方案三：腾讯云开发 CloudBase

**简介**：腾讯云提供的 Serverless 后端服务，与微信生态深度集成。

| 维度 | 评估 |
|------|------|
| 国内延迟 | ✅ 国内节点，延迟极低 |
| 稳定性 | ✅ 腾讯云基础设施 |
| 开发体验 | ⚠️ API 与 Supabase 不同 |
| 价格 | ⚠️ 基础套餐 19 元/月起 |
| 迁移成本 | ❌ 需重写后端调用代码 |

**核心功能**：
- 云数据库（文档型）
- 云函数
- 云存储
- 静态网站托管

**优势**：
- 与微信小程序深度集成
- 腾讯云生态支持

**缺点**：
- API 与 Supabase 完全不同，迁移成本高
- 用户反馈价格偏贵

**结论**：适合微信小程序项目，但当前项目迁移成本高。

**参考**：
- [腾讯云 CloudBase 官网](https://cloud.tencent.com/product/tcb)
- [CloudBase 价格文档](https://cloud.tencent.com/document/product/876/75213)

### 方案五：阿里云自建 Supabase

**简介**：在阿里云服务器上使用 Docker 或 Pigsty 自行部署 Supabase。

| 维度 | 评估 |
|------|------|
| 国内延迟 | ✅ 国内节点，延迟极低 |
| 稳定性 | ⚠️ 依赖自身运维能力 |
| 开发体验 | ✅ 完整 Supabase 功能 |
| 价格 | ⚠️ 服务器 50-100 元/月 + 运维成本 |
| 迁移成本 | ✅ 几乎为零（同 Supabase API） |

**优势**：
- 完全可控
- 功能最全（400+ PostgreSQL 扩展）
- 长期成本可能更低

**缺点**：
- 需要运维能力
- 部署配置复杂
- 需要处理备份、监控、升级等

**结论**：适合有运维能力的团队，个人开发者不推荐。

**参考**：
- [Pigsty 自建 Supabase 教程](https://pigsty.cc/blog/db/supabase/)
- [阿里云 RDS Supabase 文档](https://help.aliyun.com/zh/rds/apsaradb-rds-for-postgresql/supabase/)
- [Docker 私有化部署教程](https://www.hangge.com/blog/cache/detail_3408.html)

## 五、初步结论

### 推荐方案：MemFire Cloud

**理由**：
1. **零迁移成本**：API 100% 兼容 Supabase，只需更换 URL 和 Key
2. **国内访问快**：服务器在国内，延迟低
3. **免费起步**：个人项目免费版够用
4. **功能完整**：数据库、认证、存储、实时订阅一应俱全
5. **开发友好**：提供多种框架脚手架

### 建议开发路线

```
阶段一（当前）    阶段二（上线）      阶段三（增长）
localStorage  →  MemFire Cloud  →  按需升级/迁移
   免费            免费/低成本         付费套餐
```
