# PROJECT

这里用于放：

- 项目概览
- 当前目标
- 当前迭代范围
- 项目上下文

当前项目建议优先维护一份最小项目上下文文件，例如：

- `project-context.md`
- `研发流程与发布规范.md`
- `GitHub仓库设置操作清单.md`
- `发布Runbook.md`
- `需求开发操作手册.md`
- `.honeycomb/capabilities-local/worktree-playbook.md`

它应承接：

- 根目录 `README.md`
- 历史 `doc/02_产品定义/`
- 当前代码工程 `temporal/`

而不是继续把“总控上下文”散落在多个旧目录里。
