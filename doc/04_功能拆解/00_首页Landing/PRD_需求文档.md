# PRD: 首页 Landing Page - Phase 1.0

## 1. 功能目标

构建 **Temporal 产品首页**，作为用户的第一触点，清晰传达产品价值主张，引导用户开始使用。

- **用户价值**：让新用户在 30 秒内理解 Temporal 的核心理念——"看清时间结构，推进目标进展"
- **转化目标**：引导用户点击"开始分析我的时间"进入应用主界面

---

## 2. 设计理念

> **"时间管理 2.0"** —— 从记录工具升级为时间资产管理系统

核心信息传达：
1. **看清结构**：自动把日程与记录映射到目标与高能时段
2. **AI 赋能**：生成可执行的周报与下周排班建议
3. **目标对齐**：让忙碌变得有价值

## 4. 模块详细设计

### 4.1 导航栏 (Navbar)

**布局**：固定顶部，毛玻璃效果背景

| 元素                     | 位置 | 行为                   |
| :----------------------- | :--- | :--------------------- |
| Logo + 品牌名 "Temporal" | 左侧 | 点击返回首页           |

**导航链接**：
- 为什么选择 → `#benefits`

### 4.4 使用流程 (HowItWorks)

**标题**：闭环式时间管理

3 步流程：

| 步骤 | 标题                   | 描述                                  | 视觉展示                     |
| :--- | :--------------------- | :------------------------------------ | :--------------------------- |
| 1️⃣    | 智能排班，而非简单填空 | AI 自动避开低能时段，锁定深度工作窗口 | 拖拽任务到推荐时段的日历动画 |
| 2️⃣    | 沉浸执行，记录真实流向 | 专注模式自动屏蔽干扰，时间轴无感记录  | 计时器+时间轴记录 Mock       |
| 3️⃣    | 用数据复盘，而非凭感觉 | 系统自动生成结构化周报                | 周报+建议卡片 Mock           |

### 4.8 常见问题 (FAQ)

| 问题               | 回答                                                             |

## 5. 路由与导航逻辑

### 5.1 路由配置

| 路径    | 组件           | 说明                     |
| :------ | :------------- | :----------------------- |
| `/`     | RootDispatcher | 根据 onboarding 状态分发 |
| `/home` | LandingPage    | 首页 Landing             |
| `/app`  | ExecutePage    | 应用主界面               |

### 5.2 首页访问逻辑

```
用户访问 "/"
    ↓
检查 localStorage: temporal_has_onboarded
    ↓
已引导过 (true) → 跳转 /app (应用主界面)
未引导过 (false) → 显示 LandingPage
```
