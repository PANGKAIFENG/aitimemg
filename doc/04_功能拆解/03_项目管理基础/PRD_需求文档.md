# PRD: 项目管理基础 (Project Management Foundation) - Phase 1.5.1

## 1. 功能目标

建立 **Project (项目)** 作为一等公民实体，实现任务与项目的持久化绑定，为分屏时间轴和项目维度分析提供数据基础。

- **用户价值**：让用户的工作按项目聚焦，清晰掌握"我把时间花在了哪些项目上"。
- **AI 价值**：为 AI 按项目上下文进行智能规划提供数据基础，支持项目级别的时间预测和优先级建议。

---

## 2. 核心假设

> **用户的工作在一定周期内围绕有限的核心项目进行**。

这意味着：
1. **Project 是一等公民**：独立于泳道存在，拥有自己的生命周期。
2. **泳道是 Project 的视图投影**："打开一个泳道"本质上是"在当前视图中固定显示某个项目"。
3. **关闭泳道 ≠ 删除项目**：只是取消该项目在当前视图的固定展示，任务-项目关系不变。

---

## 3. 用户场景

### 场景 A：创建新项目 (Create Project)
> 用户开始负责"智能白板"项目，想在 Temporal 中建立这个项目。
> **操作**：在项目管理入口点击"新建项目"，输入名称和选择颜色标记。

### 场景 B：将任务归入项目 (Assign Task to Project)
> 用户创建了一个任务"设计登录页"，想把它分配到"智能白板"项目。
> **操作**：在任务详情中选择"所属项目"下拉框，选择"智能白板"。

### 场景 C：通过泳道快速归类 (Drag to Lane)
> 用户在分屏时间轴上打开了"智能白板"泳道，想把一个任务拖进去。
> **操作**：将任务从待办池拖入该泳道，任务自动关联"智能白板"项目。

### 场景 D：关闭泳道后任务仍在 (Close Lane, Task Persists)
> 用户临时关闭"智能白板"泳道，第二天重新打开。
> **预期**：之前关联的任务仍然存在，不受泳道开关影响。

### 场景 E：查看项目时间投入 (Project Analytics)
> 用户想知道本周在"智能白板"项目上花了多少时间。
> **操作**：进入统计模块，选择"按项目"维度查看。

## 6. 字段定义

### 6.1 Project 实体

| 字段名         | 类型      | 来源     | 说明                    |
| :------------- | :-------- | :------- | :---------------------- |
| **id**         | UUID      | 系统生成 | 项目唯一标识            |
| **name**       | String    | 用户输入 | 项目名称                |
| **color**      | String    | 用户选择 | 颜色标记 (Hex 或预设色) |
| **status**     | Enum      | 用户操作 | `active`, `archived`    |
| **created_at** | Timestamp | 系统生成 | 创建时间                |

### 6.2 Task 扩展

| 字段名         | 类型  | 来源     | 说明                 |
| :------------- | :---- | :------- | :------------------- |
| **project_id** | UUID? | 用户选择 | 关联的项目 ID (可选) |

## 7. 交互逻辑

### 7.1 创建项目
1. **动作**：用户点击"新建项目"。
2. **系统响应**：
    - 弹出项目创建表单 (名称 + 颜色选择)。
    - 保存后项目出现在列表中。

### 7.2 归档项目
1. **动作**：用户点击项目的 `[···]` 菜单，选择"归档"。
2. **系统响应**：
    - 项目移动到"已归档"分组。
    - **任务保留**：关联该项目的任务不删除，仍可在主轨道查看。
    - **泳道处理**：如果有泳道绑定该项目，泳道自动解绑 (变为无项目过滤)。

### 7.3 任务关联项目
1. **动作**：用户在任务详情选择"所属项目"。
2. **系统响应**：
    - 任务的 `project_id` 更新。
    - 如果当前有该项目的泳道打开，任务自动出现在该泳道中。

### 7.4 泳道绑定项目
1. **动作**：用户在泳道 Header 点击项目选择器，选择一个项目。
2. **系统响应**：
    - 泳道的 `projectId` 更新。
    - 泳道名称自动更新为项目名称 (可覆盖)。
    - 泳道内容按项目过滤。

### 7.5 关闭泳道 (重新定义)
1. **动作**：用户关闭某个绑定了项目的泳道。
2. **系统响应**：
    - **仅移除视图配置**：泳道从当前布局中消失。
    - **项目不变**：项目实体仍然存在，可随时重新打开。
    - **任务关联不变**：任务的 `project_id` 保持不变，在主轨道仍可见。

## 8. 与分屏时间轴的关系

| 原有概念         | 新定义                                           |
| :--------------- | :----------------------------------------------- |
| 泳道 (Lane)      | **视图配置**：决定当前布局显示哪些项目的任务     |
| Project          | **数据实体**：独立存储，拥有自己的 CRUD 生命周期 |
| projectId (Lane) | 指向一个已存在的 Project 实体                    |
| projectId (Task) | 任务归属的项目，持久化存储                       |

**核心变更**：
- 原 PRD 中的"泳道配置全局共享"保持不变。
- 新增"泳道可绑定已存在的 Project"。
- 关闭泳道时，任务-项目关系**不受影响**。

---

## 9. 数据兼容与迁移

| 场景                    | 处理方式                                |
| :---------------------- | :-------------------------------------- |
| **旧任务无 project_id** | 显示在主轨道，视为"未分类"任务          |
| **旧泳道配置**          | 保持原有行为 (无项目绑定时显示所有任务) |

---

## 10. 异常与边界

| 异常场景             | 表现                 | 托底方案                   |
| :------------------- | :------------------- | :------------------------- |
| **删除项目**         | 不允许删除，只能归档 | "归档"操作保留数据，可恢复 |
| **项目名称重复**     | 允许重复             | 通过颜色和 ID 区分         |
| **同一项目多个泳道** | 允许                 | 不同泳道可以绑定同一个项目 |

## 11. 验收标准

- [ ] 可以创建、编辑、归档项目。
- [ ] 任务详情中可以选择/更换所属项目。
- [ ] 泳道可以绑定一个项目，绑定后自动过滤该项目的任务。
- [ ] 关闭泳道后，任务-项目关系保持不变。
- [ ] 重新打开同一项目的泳道时，之前关联的任务仍然显示。
- [ ] (未来) 统计模块可以按项目维度查看时间投入。
