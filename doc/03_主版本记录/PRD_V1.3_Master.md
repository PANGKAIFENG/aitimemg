## 一、产品背景

### 1.1 问题洞察

现代知识工作者的时间被碎片化，任务类型复杂。其效率瓶颈不在于"缺工具"，而在于**缺乏对自身时间使用方式的认知**。

时间管理的本质不是"安排时间"，而是**理解不同任务类型与不同时间段之间的匹配规律**。为了建立这种自我节奏感，必须先对自身的时间分布进行结构化记录和导出分析。

### 1.2 产品定位

**Temporal** 是一款帮助知识工作者"认知自己时间"的轻量记录工具。

Phase 1 仅专注于：**让用户轻量地记录任务、安排时间、导出结构化数据**。

---

## 二、产品理念

| 原则 | 说明 |
|-----|------|
| **先记录 → 再导出 → 最后复盘** | 工具本身不做 AI 分析，分析交给外部 AI |
| **觉察优于提醒** | 不做自动推送，让用户自己发现模式 |
| **标签是自我沉淀** | 不是固定分类，是用户自己定义的 `【】` 格式 |
| **低摩擦记录** | 单次操作目标时长 < 10 秒 |
| **Phase 1 目标** | 帮助用户"看见自己在做什么" |

## 三、用户场景

### 3.1 用户画像

| 属性 | 描述 |
|-----|------|
| 职业 | 产品经理、设计师、创作者、自由职业者 |
| 痛点 | 不清楚时间都花在哪里，无法识别高价值时间段 |
| 目标 | 结构化记录任务，导出数据供 AI 分析复盘 |
| 行为特征 | 习惯用 `【】` 标记任务类型，重视深度工作 |

### 3.2 用户标签体系（示例）

用户可自由定义 `【任务类型-深度等级】【结果归属】` 格式的标签：

**维度一：任务类型**
| 类型 | 定义 |
|-----|------|
| 产品推进 | 方案、架构、策略设计 |
| 探索实验 | AI 新模型、新工具试验 |
| 体验打磨 | 交互细节、文案、引导设计 |
| 协同支援 | 解锁他人的沟通、对齐 |
| 评审决策 | 需求评审、优先级拍板 |
| 例会同步 | 周会、项目会、纯同步 |
| 复盘迭代 | 写复盘、抽象方法论 |

**维度二：深度等级（L1–L5）**
| 等级 | 定义 |
|-----|------|
| L1 | 简单执行：机械操作、贴票、搬运数据 |
| L2 | 简单协作：信息转发、普通会议 |
| L3 | 独立专注：写文案、小方案、集中思考 |
| L4 | 深度思考：架构设计、复杂方案 |
| L5 | 才华涌现：方法论突破、可沉淀资产 |

**维度三：结果归属**
| 归属 | 定义 |
|-----|------|
| R-OWN | 我负责的结果，直接挂在我头上 |
| R-SUP | 支撑型，帮别人推进 |
| R-NONE | 无明显结果，临时插入 |

**示例标题**：`【产品推进-L4】【R-OWN】白板传播链路讨论`

> **注意**：Phase 1 系统只做 `【】` 正则提取和高亮，不做结构化校验。

## 四、用户故事

### 4.1 核心故事

| 角色 | 故事 | 验收标准 |
|-----|------|---------|
| 知识工作者 | 我想快速创建一个任务，不中断我的心流 | Cmd+K 唤起输入框，输入标题回车即创建，< 5 秒 |
| 知识工作者 | 我想把今天要做的任务拖到时间轴上安排 | 从侧边栏或"全天"槽拖入时间轴，自动吸附到 15 分钟刻度 |
| 知识工作者 | 我想看到任务超时了没完成 | 超时任务卡片边框变红，显示延期时长 |
| 知识工作者 | 我想快速标记任务完成 | 点击卡片上的复选框，卡片变灰 + 删除线 |
| 知识工作者 | 我想用自己的标签体系记录任务类型 | 支持 `【】` 格式，输入时实时高亮 |
| 知识工作者 | 我想导出本周数据给 AI 分析 | 点击菜单 → 导出 CSV，文件可被 AI 解析 |
| 知识工作者 | 我想看到今天的安排是否过满 | 顶部显示容量进度条，> 100% 时变红 |

### 4.2 边缘故事

| 角色 | 故事 | 验收标准 |
|-----|------|---------|
| 知识工作者 | 我想查看昨天的时间安排 | Header 左右箭头切换日期 |
| 知识工作者 | 我想删除一个不再需要的任务 | 编辑弹窗点击"不再关注"，任务消失 |
| 知识工作者 | 我想调整任务的时长 | 拖拽卡片底部边缘，实时显示新时长 |

## 七、数据结构

### 7.1 tasks 表

| 字段 | 类型 | 约束 | 说明 | Phase 1 |
|-----|------|------|------|--------|
| id | UUID | PK | 任务唯一 ID | ✅ |
| title | VARCHAR | NOT NULL | 任务标题（含 `【】` 标签） | ✅ |
| notes | TEXT | | Markdown 格式备注 | ✅ |
| status | ENUM | | `todo` / `done` / `abandoned` | ✅ |
| created_at | TIMESTAMP | DEFAULT NOW | 创建时间 | ✅ |
| updated_at | TIMESTAMP | | 更新时间 | ✅ |
| tags | TEXT[] | | 提取的标签数组 | 预留 |
| deleted_at | TIMESTAMP | | 软删除时间（回收站） | 预留 |
| week_start_date | DATE | | 归属周起始日 | 预留 |
| parent_id | UUID | FK | 父任务 ID（子任务） | 预留 |

### 7.2 schedule_events 表

| 字段 | 类型 | 约束 | 说明 | Phase 1 |
|-----|------|------|------|--------|
| id | UUID | PK | 排班事件 ID | ✅ |
| task_id | UUID | FK | 关联任务 | ✅ |
| date | DATE | NOT NULL | 排班日期 | ✅ |
| planned_start | FLOAT | 0-24 | 计划开始（如 14.5 = 14:30） | ✅ |
| planned_duration | FLOAT | MIN 0.25 | 计划时长（小时） | ✅ |
| is_all_day | BOOLEAN | DEFAULT false | 是否在"全天"槽 | ✅ |
| actual_done_at | TIMESTAMP | | 实际完成时间 | 预留 |
| offset_min | INT | | 计划与实际偏差分钟 | 预留 |

### 7.3 导出 CSV 字段（v1.3）

task_id,title,status,date,planned_start,planned_end,duration_min,notes,created_at
