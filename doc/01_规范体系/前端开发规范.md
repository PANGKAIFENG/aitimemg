# 前端开发规范 (L2)

> **目标**：确保 Temporal 的 UI 呈现出统一的、高规格的 AI Native 审美。

## 1. UI & 设计语言
*   **风格关键词**：Pastel (柔和色系)、Minimalist (极简)、Glassmorphism (毛玻璃)、Soft Shadows (柔和阴影)。
*   **色系约束**：
    *   必须优先使用 `src/index.css` 中定义的 CSS 变量。
    *   **禁止**在组件内直接使用硬编码色值（如 `#3b82f6`）。
*   **组件库**：
    *   本项目优先使用 **原生 React 组件 + 原生 CSS** 进行高定制开发。
    *   避免引入重量级 UI 框架，保持轻量。
*   **图标**：统一使用 `lucide-react` 图标库。

## 2. 组件复用原则
在开发任何新组件前，**必须**检查以下现有组件是否可复用或扩展：
*   `src/components/common/Button.tsx`: 统一按钮样式。
*   `src/components/modals/ConfirmModal.tsx`: 弹窗基础。
*   `src/components/TaskCard.tsx`: 时间轴核心卡片。

## 3. CSS 编写规范
*   **主选方案**：原生 CSS (配合 CSS Modules 或 `src/index.css` 变量)。
*   **样式层级**：全局变量 > 组件局部 CSS > Inline Styles (极少)。
*   **视觉还原度**：代码实现必须 **100% 还原** PRD 中的 UI 设计稿。如果没有设计稿，必须要求 AI 生成。

## 4. 验证要求
*   UI 变更必须配套截图或录屏。
*   必须对照 `TEST.md` 中的交互点进行自测。

## 5. 安全与防退化 (Safety First)
> 详细机制请强制阅读：**[质量保障机制](./质量保障机制.md)**

*   **防御性编程**：
    *   外部依赖必须做非空/异常检查。
    *   禁止使用 `any`，类型导入强制使用 `import type`。
*   **架构隔离**：
    *   单文件超过 300 行必须考虑拆分。
    *   复杂逻辑（拖拽、计算）必须抽离为纯函数 Utils 或 Custom Hooks，并配套单元测试。
*   **原子化提交**：
    *   严禁在 Feature 分支中夹带 Refactor 代码。
