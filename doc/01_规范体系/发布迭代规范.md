# Temporal 发布迭代规范 (Release Standards)

> 📌 **目的**: 规范项目的版本管理、分支管理与发布流程，确保线上环境 (`aitimemg.cn`) 的稳定性与可追溯性。

---

## 1. 版本管理 (Versioning)

遵循 [Semantic Versioning 2.0.0](https://semver.org/) 规范：

**格式**: `Major.Minor.Patch` (主版本号.次版本号.修订号)
- **Major (主版本号)**: 做了不兼容的 API 修改或重大架构重构 (e.g., v1.0.0 -> v2.0.0)。
- **Minor (次版本号)**: 做了向下兼容的功能性新增 (e.g., v1.0.0 -> v1.1.0)。
- **Patch (修订号)**: 做了向下兼容的问题修正 (e.g., v1.0.1 -> v1.0.2)。

**示例**:
- `v0.5.0`: 当前 MVP 版本。
- `v0.5.1`: 修复了 UI 样式问题。
- `v0.6.0`: 新增了 "AI 全自动执行" 功能。

## 2. 分支管理 (Branching Model)

| 分支名称      | 说明                                                  | 保护策略                                           |
| :------------ | :---------------------------------------------------- | :------------------------------------------------- |
| **main**      | **生产主分支**。对应线上环境 (`aitimemg.cn`) 的代码。 | **禁止直接 Push**，仅允许通过 Merge Request 合并。 |
| **develop**   | (可选) 开发主分支。集成了所有待发布的特性。           | 允许 commit，定期合并到 main。                     |
| **feature/*** | **特性分支**。用于开发新功能。                        | 开发完成后合并回 main 或 develop。                 |
| **fix/***     | **修复分支**。用于修复 Bug。                          | 修复完成后合并回 main。                            |

**推荐工作流**:
1. 从 `main` 拉取新分支 `feature/new-task-modal`。
2. 在本地开发、调试 (localhost:5173)。
3. 提交代码。
4. 测试通过后，合并回 `main`。

---

## 3. 发布流程 (Release Workflow)

当准备将本地改动发布到线上域名时，请严格执行以下步骤：

### 阶段一：发布准备 (Preparation)
1.  **代码冻结**: 确认所有功能已开发完毕，不再接收新功能提交。
2.  **本地构建检查**:
    ```bash
    cd temporal
    npm run build
    # 确保无报错，且生成了 dist 目录
    ```
3.  **版本号更新**:
    -   更新 `package.json` 中的 `version` 字段。
    -   (可选) 更新 `CHANGELOG.md` 记录本次变动。

### 阶段二：正式发布 (Deployment)
1.  **执行部署**:
    ```bash
    npm run deploy
    ```
    *(注：此命令会将 dist 目录上传至阿里云 OSS)*
2.  **Git 打标签**:
    ```bash
    git commit -am "chore: release v0.5.x"
    git tag v0.5.x
    git push origin v0.5.x
    ```

### 阶段三：线上验证 (Verification)
1.  访问 [https://aitimemg.cn/app](https://aitimemg.cn/app)。
2.  强制刷新缓存 (Cmd+Shift+R)。
3.  验证核心功能及本次修复点。

---

## 4. 回滚机制 (Rollback)

如发现线上有重大 Bug，需立即回滚：

1.  **OSS 回滚**: 阿里云 OSS 控制台支持开启版本控制，可恢复到上一个 Object 版本（需提前开启）。
2.  **重新发布旧版本**:
    -   本地切回上一个 tag: `git checkout v0.5.0`
    -   重新构建发布: `npm run build && npm run deploy`

---

## 5. 附录：发布检查清单 (Release Checklist)

每次发布前请在脑海中或文档中勾选：

- [ ] `npm run dev` 本地运行无报错。
- [ ] `npm run build` 本地构建无报错。
- [ ] 关键功能（如登录、任务创建）已在本地验证通过。
- [ ] 环境变量 (`.env.local`) 确认指向生产环境配置。
- [ ] 确认已备份重要数据（如果涉及数据库变更）。
