## 2. 文档类型定义

| 文档类型            | 命名格式                | 存放位置           | 目标读者 | 更新频率       |
| ------------------- | ----------------------- | ------------------ | -------- | -------------- |
| **项目总入口 (L1)** | `README.md`             | 项目根目录         | 全员、AI | 架构调整时     |
| **规范体系 (L2)**   | `*.md`                  | doc/01_规范体系/   | 开发、AI | 规范变更时     |
| **PRD生成规范**     | `产品PRD生成规范.md`    | doc/01_规范体系/   | PM、AI   | PRD标准变更时  |
| **主版本记录**      | `PRD_V{X.X}_Master.md`  | doc/03_主版本记录/ | PM、开发 | 极重大版本时   |
| **功能包 (Bundle)** | 文件夹：`[序号]_[名称]` | doc/04_功能拆解/   | 开发、AI | 每个具体需求   |
| **PRD (需求+UI)**   | `PRD_需求文档.md`       | doc/04_功能拆解/库 | PM、开发 | 每个功能包必备 |
| **TS (技术方案)**   | `TS_技术方案.md`        | doc/04_功能拆解/库 | 开发     | 逻辑变更时必备 |
| **TEST (测试用例)** | `TEST_测试用例.md`      | doc/04_功能拆解/库 | 开发、QA | 验收自测时必备 |
| **CHANGELOG**       | `CHANGELOG.md`          | temporal/          | 用户     | 版本发布时     |

## 4. 技术文档内容定义

`需求文档/技术文档/` 目录专门用于存放非业务需求类的技术资料。主要包含：

1.  **架构设计** (`架构设计.md`)：系统的整体分层、模块交互、核心流程图。
2.  **数据库设计** (`数据库设计.md`)：ER 图、表结构定义、索引设计。
3.  **API 接口定义**：如果前后端分离，记录核心接口契约。
4.  **难点攻关记录**：记录复杂技术问题的解决方案 (Troubleshooting)。

### 4.1 架构/技术方案模板

```markdown
# TS_技术方案: [功能名称]

## 1. 设计背景
[为什么需要这个设计？涉及哪些模块？]

## 2. 总体实现思路
[组件拆分 / 逻辑分层 / 状态流转]

## 3. 详细设计
### 3.1 接口/数据结构变更
[API 定义 / JSON 示例 / DB 字段]

### 3.2 核心逻辑实现
[流程图或伪代码说明]
```

## 6. Git Commit 规范

**格式**（Conventional Commits）：
```
<type>(<scope>): <description>
```

**Type 类型**：
| 类型       | 说明      | 示例                            |
| ---------- | --------- | ------------------------------- |
| `feat`     | 新功能    | feat(timeline): 卡片布局优化    |
| `fix`      | Bug 修复  | fix(drag): 修复拖拽不可见       |
| `docs`     | 文档      | docs: 更新文档规范              |
| `refactor` | 重构      | refactor(service): 优化代码结构 |
| `style`    | 样式      | style: 调整间距                 |
| `chore`    | 构建/工具 | chore: 更新依赖                 |

## 8. 工作流程

```
新需求 → 创建 PRD → 开发 → Git Commit → 更新迭代记录表 → (发布时) 更新 CHANGELOG
```

| 阶段         | 动作                               |
| ------------ | ---------------------------------- |
| 需求确认     | 创建 PRD 文档                      |
| 开发中       | 按规范写 Commit Message            |
| 开发完成     | 更新迭代记录表（关联 PRD、Commit） |
| 版本发布     | 更新 CHANGELOG                     |
| 重大架构变更 | 更新架构设计文档                   |
