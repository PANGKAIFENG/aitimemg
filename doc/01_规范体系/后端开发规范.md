# 后端开发规范 (L2)

> **目标**：确保 Temporal 的数据存取层、API 设计具备高鲁棒性与类型安全性。

## 1. API 调用规范
*   **归口管理**：所有外部接口调用或 Mock 数据层必须封装在 `src/services/` 目录下。
*   **契约先行**：每一个核心 Service 必须有对应的接口文档，记录在功能的 `TS.md` 文档中。

## 2. 类型安全 (Type Safety)
*   **强制类型定义**：所有业务对象（Tasks, Schedules, Users）必须在 `src/types/` 下有明确的 TypeScript 定义。
*   **禁止 any**：项目中 **严禁** 使用 `any` 类型，除非是不得不适配极其不规范的第三方库。

## 3. 数据存取逻辑
*   业务逻辑应尽量与 UI 渲染分离。
*   复杂的数据计算、格式转换建议放在 `src/utils` 或对应的 Store 逻辑中。

## 4. 注意事项
*   涉及数据库 Schema 变更（例如新增 `executor` 字段），必须先更新项目的“数据库设计”文档或在功能的 `TS.md` 中详细说明迁移逻辑。
