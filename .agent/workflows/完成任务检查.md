# 任务完成检查流程

在完成任何功能开发、优化或 Bug 修复后，**必须**执行以下步骤：

## 1. 检查 PRD 文档
- [ ] 如果是新功能/优化，确认已在 `需求文档/PRD/` 创建 `PRD_功能名称.md`
- [ ] 如果是已有 PRD 的迭代，确认已更新版本记录

```bash
# 查看最近修改的 PRD
ls -lt 需求文档/PRD/ | head -5
```

## 2. 更新迭代记录表
- [ ] 在 `需求文档/00_迭代记录表.md` 添加一行记录
- [ ] 包含：日期、版本、类型、描述、PRD 链接

```bash
# 查看最新迭代记录
head -20 需求文档/00_迭代记录表.md
```

## 3. 更新 CHANGELOG
- [ ] 在 `temporal/CHANGELOG.md` 当前版本下添加变更
- [ ] 如果是新版本，创建新的版本块

```bash
# 查看最新 CHANGELOG
head -25 temporal/CHANGELOG.md
```

## 4. 最终确认
执行以下命令确认所有文档已更新：

```bash
# 查看今天修改的文档文件
find . -name "*.md" -mtime -1 -type f | grep -E "(PRD|迭代|CHANGELOG)" | head -10
```

---

> ⚠️ **重要**：在向用户报告「完成」之前，必须完成上述所有检查！
